01Premise

This guide is the IT operations reference for deploying, supporting, and maintaining the deccan-design v2.0 system across the Deccan Fine Chemicals fleet — Windows endpoints managed via ManageEngine Endpoint Central and Microsoft Intune, macOS endpoints managed via Jamf, plus the Google Workspace tenant and the Claude.ai accounts of individual users.

The system is documented in deccan-design-spec.html and demonstrated in deccan-design-specimen.html. This guide is operational: who, what, when, and how to respond when something does not work.

02Rollout plan

Four waves. Each wave's entry criteria are the previous wave's exit criteria. Rollback is reversible at every stage — the installer is per-user and removable via standard MSI / pkgutil uninstall.

WaveCohortSizeEntry criteriaExit criteria
PilotIT users5Repository released; MSI and PKG built.All five report templates in Word/Excel/PowerPoint Personal gallery; Outlook signature visible after helper script; Claude skill enumerated; no blocker tickets.
Wave 1Engineering & R&D50Pilot exit criteria met. Help-desk runbook posted (this guide).≥ 90% of cohort installed successfully without escalation. SmartScreen / Gatekeeper FAQ used by ≤ 20%. Zero P0 / P1 incidents.
Wave 2Office users300Wave 1 exit criteria met. Comms email sent (preferences + admin-guide link).≥ 95% of cohort installed. Outlook signature configured by ≥ 80%. Personal preferences set by ≥ 60%.
FullRemaining fleet~1,800Wave 2 exit criteria met. Onboarding documentation updated for new hires.≥ 95% of fleet installed. The next forty Deccan documents issue under v2.0 without per-document deviation.

Rollback

Per-wave rollback procedure:

  1. Pause distribution in Endpoint Central / Intune / Jamf for the cohort.
  2. Push an uninstall command to the cohort (msiexec /x <ProductCode> /qn on Windows; pkgutil --forget + filesystem removal on macOS).
  3. Notify the cohort via email with the rollback rationale and the timeline for re-deployment.
  4. Document the rollback trigger in the incident log; review against waves before resuming.

03Deployment via ManageEngine Endpoint Central

Endpoint Central handles the majority of the Deccan Windows fleet. The MSI is per-user, so the package definition runs in the user context rather than system context.

Configuration repository

  • Upload deccan-design-2.0.0.msi to the Endpoint Central package repository.
  • Tag with: deccan-design, v2.0.0, user-context, unsigned.
  • Set a checksum (SHA-256) on upload — verify the same checksum is published on the GitHub release.

Package definition

  • Install command: msiexec /i deccan-design-2.0.0.msi /qn
  • Uninstall command: msiexec /x {ProductCode} /qn (where {ProductCode} is the value emitted at MSI build time).
  • Detection rule: registry key HKCU\Software\deccan-design\Version equals 2.0.0.
  • Reboot: not required.
  • Elevation: not required (per-user install).

Distribution rules

  • Pilot: target by username, five named users.
  • Wave 1: target by AD group cn=eng-rd,ou=groups.
  • Wave 2: target by AD group cn=office-users,ou=groups.
  • Full: target by AD group cn=domain-users,ou=groups minus the previous waves.

Self-service portal

Add the package to the Endpoint Central self-service portal under the "Productivity tools" category for the pilot wave, so pilot users can opt-in rather than have the install pushed silently. Wave 1 onwards uses the silent push.

04Deployment via Microsoft Intune

For endpoints managed via Intune (typically the laptop-mobile fleet), the MSI is wrapped as a Win32 app via IntuneWinAppUtil.

Wrapping

IntuneWinAppUtil.exe `
  -c <source-folder-containing-msi> `
  -s deccan-design-2.0.0.msi `
  -o <output-folder>

Produces deccan-design-2.0.0.intunewin.

Win32 app registration

  • App information: name "deccan-design v2.0"; publisher "Deccan Fine Chemicals"; description sourced from the repository README.
  • Program — install command: msiexec /i deccan-design-2.0.0.msi /qn
  • Program — uninstall command: msiexec /x {ProductCode} /qn
  • Install behaviour: User (per Decision 4 in PRD §1.4).
  • Device restart behaviour: No specific action.

Detection rules

Use two rules in an AND combination:

  1. MSI product code — the product code emitted at MSI build time (read from the WiX output).
  2. File existence — %APPDATA%\Microsoft\Templates\deccan-document.dotx exists.

Assignment groups

  • Pilot: Azure AD group deccan-design-pilot.
  • Wave 1: deccan-design-wave1.
  • Wave 2: deccan-design-wave2.
  • Full: deccan-design-all, minus previous waves.

Supersedence

When v2.0.1 (or later) ships, mark v2.0.0 as superseded in Intune. Intune handles the in-place upgrade — the MSI's MajorUpgrade element does the actual work of removing v2.0.0 before installing the new version.

05Deployment via Jamf for macOS

The Jamf-managed macOS fleet receives the .pkg via policy.

Policy configuration

  • Upload deccan-design-2.0.0.pkg to the Jamf distribution point.
  • Create a policy Install deccan-design v2.0 with the package, install action "install", and frequency "once per computer".
  • Set the trigger: "Recurring check-in" for the silent-deploy waves; "Self Service" for the pilot.
  • Add a maintenance action: "Update inventory" after install.

Smart group

Create a smart group deccan-design installed with criterion "Application Title is deccan-design" or "File exists at ~/Library/Application Support/deccan-design". Use the smart group for inventory.

Scope

Expand scope across waves as on Windows: pilot users → engineering / R&D smart group → office-user smart group → all-Mac smart group.

06Unsigned-installer support runbook

This is the single most likely source of pilot help-desk tickets. The MSI and PKG ship unsigned per PRD §1.4 Decision 5. Three concrete user-symptom sequences with diagnostic and remedy steps.

Windows SmartScreen — "Windows protected your PC"

Symptom. User double-clicks the MSI from File Explorer or a Teams download. SmartScreen appears with the wording above and a single visible button "Don't run". A subtle "More info" link is present but easy to miss.

Diagnostic. No diagnostic needed — this is expected behaviour for an unsigned installer.

Remedy.

  1. Tell the user: "Click the More info link in the dialog."
  2. A new button Run anyway appears under the program name.
  3. Click Run anyway. The installer proceeds normally.

Escalation. If the user is reluctant to dismiss the warning, IT can pre-stage the MSI via Intune in silent mode (msiexec /i ... /qn), which bypasses SmartScreen for the user entirely. This is the recommended path for Wave 1 onwards.

Windows Defender Application Control quarantine

Symptom. On endpoints with stricter WDAC policies (typically Restricted-classification machines and lab workstations), the MSI is quarantined silently — the user reports "nothing happened when I double-clicked."

Diagnostic. Check Defender for Endpoint's quarantine view filtered by the user's endpoint and the MSI's SHA-256. If the MSI is present, this is the cause.

Remedy.

  • Release the file from quarantine via Defender for Endpoint (admin action), or
  • Add the MSI hash to the WDAC allowlist for the duration of the rollout, then remove it post-deployment.

macOS Gatekeeper — "cannot be opened because Apple cannot check it for malicious software"

Symptom. User double-clicks the .pkg from Finder. Gatekeeper blocks with the wording above.

Diagnostic. No diagnostic needed — expected behaviour for an unsigned .pkg.

Remedy.

  1. Right-click the .pkg in Finder → Open → confirm in the secondary dialog.
  2. Or: System Settings → Privacy & Security. Scroll to the bottom and click Open Anyway next to the blocked-app notice. The user is prompted for their password.

Escalation. For Jamf-managed endpoints, the silent policy push bypasses Gatekeeper entirely. This is the recommended path for Wave 1 onwards.

07Outlook signature configuration

The installer drops the unconfigured signature template into %APPDATA%\Microsoft\Signatures\deccan-signature-template.htm (Windows) or the equivalent macOS path. A helper script configures it.

Windows

& "$env:APPDATA\deccan-design\Set-DeccanSignature.ps1"

The script prompts for Name, Role, Email, and Phone. Writes Deccan.htm and Deccan.txt into the Signatures folder.

macOS

~/Library/Application Support/deccan-design/set-deccan-signature.sh

Same four prompts; writes the configured signature to the Outlook signatures directory in the new Outlook for Mac container.

Selecting the signature in new Outlook

  1. Open new Outlook.
  2. Settings (gear icon) → Accounts → Signatures.
  3. Confirm "Deccan" appears in the list.
  4. Set as default for "New messages" and "Replies/forwards".

Troubleshooting

  • Signature missing from compose dropdown → check %APPDATA%\Microsoft\Signatures\Deccan.htm exists. If not, the helper script was not run.
  • Placeholders ({{NAME}} etc.) appear in the sent email → the helper was not run, or it ran against the template instead of writing Deccan.htm. Re-run the helper.
  • Signature renders without colour in Gmail recipients → expected; some recipient mail clients strip background colour. The fallback plain-text signature is in Deccan.txt.

08Personal preferences rollout

The installer cannot push Claude.ai preferences server-side. Each user pastes the text from claude/personal-preferences.md into Claude.ai → Settings → Profile → Preferences.

IT operations checklist

  • Send a comms email to each rollout cohort 24 hours before the silent install. Include: the preferences text, the URL https://claude.ai/settings, and a sentence on why the step is manual (Claude.ai accounts are user-authenticated; the server has no API for IT to push).
  • Update the new-hire onboarding documentation to include the preferences step in the first-day Claude.ai setup.
  • Add a tile to the IT self-service portal: "Configure my Claude.ai preferences" with the text and the link.

09Support runbook (general)

Symptom → diagnostics → fix

Template not appearing in Word's New → Personal gallery

  • Diagnostic: check HKCU\Software\Microsoft\Office\16.0\Common\General\PersonalTemplates. Should equal %APPDATA%\Microsoft\Templates.
  • Diagnostic: confirm %APPDATA%\Microsoft\Templates\deccan-document.dotx exists.
  • Fix: repair the MSI via Settings → Apps → deccan-design → Modify → Repair. Or reinstall.

Outlook signature missing

  • Diagnostic: check %APPDATA%\Microsoft\Signatures\Deccan.htm exists.
  • Fix: if missing, the helper script was not run. Run Set-DeccanSignature.ps1.

Claude Code does not enumerate the deccan-design skill

  • Diagnostic: confirm %APPDATA%\Anthropic\Claude\skills\deccan-design\SKILL.md exists.
  • Diagnostic: run the Claude Code skill enumeration command. If the skill is listed but does not activate, check the SKILL.md frontmatter.
  • Fix: reinstall, or copy the skill directory from the repository manually if local-only override is preferred.

Fonts appear wrong on Windows 10

  • Diagnostic: expected behaviour. Segoe UI Variable's Display axis does not resolve on Windows 10; the chain falls back to static Segoe UI.
  • Fix: none required. The fallback is metric-compatible and visually acceptable.

Document looks correct in Word but wrong in emailed PDF

  • Diagnostic: user is printing to PDF via a non-standard driver (Adobe PDF Printer, Microsoft Print to PDF with custom settings, etc.) that substitutes fonts.
  • Fix: instruct File → Export → Create PDF/XPS Document. This embeds the OS-resident font and produces a faithful PDF.

10Known issues

  • Segoe UI Variable Display axis on Windows 10. Does not resolve; falls back to static Segoe UI. Visually acceptable. Not a defect.
  • Cascadia Mono programming ligatures. Off by document default to keep ligatures out of prose. Documents that want ligatures can enable them per-document with font-feature-settings: "calt", "liga";.
  • DOCX end-page vertical centring. Uses a spacer-paragraph workaround because Word's vertical alignment center at section level affects only the running pages, not the section's last page. The visual effect is identical.
  • Unsigned installer triggers SmartScreen / Gatekeeper on every first install. Per Decision 5. Silent-deploy via Intune / Jamf bypasses the warning. See §06.
  • New Outlook for Mac signature folder location varies. Microsoft has changed the location twice since the Outlook-on-Mac rewrite. The helper script targets the current location as of May 2026; if Microsoft moves the folder again, update set-deccan-signature.sh.

11FAQ

Why is the installer unsigned?

The cost of code-signing was traded against the cost of per-endpoint dismissal friction. Signing requires certificate procurement, ongoing PKCS#12 management, and a per-build signing step. The trade is documented in PRD §1.4 Decision 5; the dismissal procedure is in §06 above.

Why is the type stack OS-native instead of a bundled font?

To avoid an MSI font deployment to every endpoint, an Intune compliance lane to verify the install, an M365 Office Font Service dependency, and a commercial font-licensing programme. The OS-native chain resolves to Segoe UI Variable on Windows 11 and San Francisco on macOS — both editorial-quality variable fonts.

Why no PDFs in the repository?

PDFs sandbox-rendered without the resident corporate fonts would substitute typefaces and confuse readers. PDF production is shifted to the user, who renders on a managed endpoint with the resident fonts available. The README documents the two methods.

What if I want to use a different font for a specific document?

You do not. The system is the contract — documents that override the type stack fail review. If the system needs to change, raise the request with the design-system owner per the escalation matrix.

What about classic Outlook?

Out of scope per PRD §1.4 Decision 6. New Outlook only.

Does this work on Google Workspace?

Yes. The Workspace templates in templates/gworkspace/ are .docx / .xlsx / .pptx mirrors of the native Office templates. Workflow: upload + open with Google Docs/Sheets/Slides, then save as Google native. The Gmail signature is a separate paste-into-settings snippet.

How do I uninstall?

Windows: Settings → Apps → deccan-design → Uninstall. macOS: see installers/macos/README.md — runs pkgutil --forget plus filesystem removal.

How do I migrate a document written under swiss_design_at_deccan v1.0 (IBM Plex)?

Replace the font-family declaration with the v2.0 chain. Colour tokens, spatial system, components, and tone rules are unchanged. See §11 of the spec.

How do I migrate a document written under deccan-design v1.0 (Aptos)?

Replace the font-family declaration with the v2.0 chain. Remove any Aptos override. See §11 of the spec.

What if Microsoft changes the new Outlook signature folder again?

Update installers/windows/Set-DeccanSignature.ps1 and installers/macos/set-deccan-signature.sh with the new path, increment the system version to 2.0.x, and re-deploy. The helper scripts isolate this concern from the rest of the system.
