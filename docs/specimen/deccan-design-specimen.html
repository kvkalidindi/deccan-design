Plate 02

The system at a glance

A document standard for Deccan Fine Chemicals: editorial-quality typography on the OS-native font stack, a single accent at four opacity stops, a ten-stop neutral grayscale, swiss discipline, and a small set of components that earn their keep.

The version-two refinement keeps everything from the original swiss_design_at_deccan that was working — the colour tokens, the spatial grid, the component vocabulary, the tone-and-voice rules — and replaces only the type stack. The previous IBM Plex requirement is dropped. The deferred Aptos plan is dropped. The new primary face is whatever the operating system already provides: Segoe UI Variable on Windows 11, San Francisco on macOS, the platform default everywhere else. No font binaries, no MSI deployment, no Microsoft 365 dependency.

The aesthetic that follows is editorial-corporate: type with rhythm at every size, colour that behaves like ink rather than paint, components that disappear when the content is the point and step forward when the structure is. The plates that follow demonstrate each surface in sequence.

The lead paragraph at --fs-lead in --stone-700; body paragraphs at --fs-body in --stone-800; measure held at 70 characters throughout.

Plate 05

Body type and reading rhythm

A document is read, in the end, as paragraphs. The system optimises for sustained reading at the densities of the corporate fleet — 1080p and 1440p external monitors, 11-inch laptop panels at 1200p — and for high-quality print at 300 DPI. The body, lead, and small sizes are tuned for that envelope.

Editorial reading depends on three things working together: a body size large enough to read at arm's length without leaning forward; a line-height generous enough that the eye can return to the next line without losing the column; and a measure short enough that the eye does not drift while tracking left-to-right. The system sets these at 17 pixels, 1.65, and 70 characters respectively, on screen. In print the same three constants resolve to 10.5 points, 1.5, and the live content area at 0.8-inch margins, which produces the same visual rhythm.

Inline emphasis exists in three registers. Strong emphasis in semibold draws the eye to a noun or claim. Italics mark titles, technical terms on first use, and the occasional voice-of-the-author qualifier — used sparingly because italics, in a sans-serif system, fight the type discipline if overused. Inline code chips in monospace on a stone-100 chip identify file paths, commands, function names, and protocol tokens — the elements where the wire-level form of the word is part of the meaning. Hyperlinks, when present, take their colour from the accent — visit the v2.0 specification — with a 30-percent-opacity underline that strengthens on hover.

Reading rhythm benefits from variation. A long sequence of body paragraphs with no structural marker is harder to enter and harder to leave; the eye has nothing to anchor on. The pull-quote, the callout, and the figure all serve as anchors for the eye and as deliberate rest stops in the argument. The pull-quote earns its place once or twice in a long section; the callout earns its place wherever the rule, the warning, or the editorial aside is genuinely a separate object from the body.

Rendered above in Segoe UI Variable Text on Windows, San Francisco on macOS, the platform default elsewhere. Optical sizing automatically selects the Text variant for body copy.

Plate 06

Lists, definitions, and emphasis

Bulleted list

  • The list-item marker tints to --stone-500 via ::marker, sitting visually quieter than the body text it leads.
  • List items inherit the body line-height and bottom margin; the spacing between items reads as deliberate rather than accidental.
  • Nested lists indent on a 28-pixel hanging indent, with a smaller top margin so that nesting reads as one continuous structure.

Numbered list

  1. Numbered lists use the default decimal marker — preferred over Roman or alpha because Roman numerals strain the eye at corporate-document scale.
  2. The marker tint matches the bulleted list. Numbering is for ordering; the visual emphasis stays on the body.
  3. Numbered lists are reserved for genuinely ordered content — sequences, ranked options, procedural steps. Unordered content uses bullets.

Definition list

Hash pinning
Recording a SHA-256 hash alongside every package version in a dependency manifest, so that the installer refuses any installation whose hash does not match the recorded value.
Internal mirror
A package server that sits between PyPI and Deccan endpoints, caching approved wheels, enforcing an allowlist, and producing a complete audit trail of every package every workstation has ever requested.
Plant-zone DMZ
A corporate-side network segment at a manufacturing plant, separated from the OT (operational-technology) network by a data-diode boundary, where analyst workstations operate on read-only OT historian extracts without OT-network access.

Definition lists pair a semibold term with a stone-200-bordered description block; the most legible glossary treatment in the system.

Plate 07

Code · the mono-and-stone discipline

The system has one rule for monospace type: mono and stone always travel together. Inline mono on a white background reads as a typo; stone-100 on sans body reads as a callout. Together they read as code, which is the only thing they should ever read as. The rule applies to file paths, command names, environment variables, function names, JSON tokens — the wire-level forms of the language whose exact characters are part of the meaning.

Inline chip

The standard primary interpreter is installed at %LOCALAPPDATA%\Programs\Python\Python312, with project virtual environments at %USERPROFILE%\projects\<name>\.venv. Run uv sync to resolve a lockfile, jupyter lab to start the notebook server, and pip-audit to scan for known CVEs. Configure JUPYTER_CONFIG_DIR only when the default location is unsuitable.

Block

  # The supported dependency-management invocation
  $ cd ~/projects/chemprop-pilot
  $ uv venv
  $ uv pip compile requirements.in -o requirements.txt --generate-hashes
  $ uv pip sync requirements.txt
  $ pip-audit -r requirements.txt

The block uses the same colour pair at scale: stone-100 fill, mono type, no border. It holds the body measure on screen and breaks to full content width in print where margins already provide the constraint.

Mono types out of the system stack — Cascadia Mono on Windows 11, SF Mono on macOS, Consolas on older Windows, the generic monospace fallback elsewhere. Programming ligatures are off in the document default; Cascadia Code with ligatures is available where intentional.

Plate 08

Callouts and the pull quote

Two surfaces for content that wants to step out of the body argument: the callout, for rules and warnings and editorial asides; the pull quote, for editorial rhythm in long-form writing.

Callout, default

Provisioning rule

Install CPython 3.12 from the official python.org Windows installer or its macOS equivalent, pinned to the latest patch release at provisioning time. System Python supplied by a Linux distribution is not the standard. The active development interpreter on any Deccan-managed Linux server is the explicitly-installed CPython build.

Callout, prohibition

Prohibited

The Anaconda Distribution Installer, because Deccan exceeds the 200-employee Anaconda commercial-license threshold and holds no commercial license. Default-channel Miniconda — that is, configured to pull from repo.anaconda.com — for the same reason. Personal Python installations not deployed through Intune on Deccan-managed devices.

Callout, muted

Discipline note

The muted variant exchanges the Deccan Blue left rule for a stone-700 rule. Used for asides about the system itself — discipline reminders, internal notes, editorial commentary — that should not draw the eye to the brand accent. The muted callout reads as quiet by design.

Pull quote

Security controls that change a user's commands fail in practice. Security controls that move policy into the infrastructure succeed.

v2.0 spec · §06 components · callout, key

The pull quote earns its place once or twice in a long section; it is a rest stop, not a decoration. The cite line in mono caps anchors it to its source.

Plate 09

Tables · the underline-and-band convention

The system has one table treatment: mono caps headers on a stone-100 fill, with a 2-pixel solid stone-900 underline; banded body rows alternating between paper and stone-50; 1-pixel stone-200 row separators; no column borders. The eye uses the band and the separator to track horizontally; column borders add noise the table does not need.

Plate 10

Sidenotes, folios, page furniture

Two editorial conventions that distinguish a finished document from a draft: sidenotes for content that supports but does not interrupt the argument, and a deliberate folio treatment for the running footer.

Sidenotes

The standard primary interpreter is CPython 3.12, with Python 3.13 as an approved alternative. Python 3.12 carries full ecosystem coverage in the anticipated Deccan stack.1 Its support runway extends through October 2028. Python 3.14, released in October 2025, is deferred as the corporate baseline because some chemistry and ML libraries on the CRDMO roadmap still resolve cleanly only on 3.12 or 3.13.2

The version landscape is reviewed quarterly. The next review of the deferred-3.14 position is November 2026.

[1] Anticipated stack: NumPy, pandas, SciPy, scikit-learn, PyTorch, TensorFlow, RDKit, Chemprop, AVEVA-related Python clients, JupyterLab, and the Anthropic SDK. All ship pre-built wheels for 3.12 on Windows, macOS, and Linux as of May 2026.
[2] Notable holdouts on 3.12/3.13: BoTorch (3.12 only), specific RDKit-derivative ML libraries, two internal AVEVA bridges. The list shortens monthly.

Running footer · folio

The convention: mono, 8.5pt, stone-500. Left-aligned imprint, right-aligned bare-integer page number. No "Page 12 of 47" wrapper — the integer is the convention. Suppressed on the cover and the end page.

Plate C · Tone and voice

Three before / after rewrites

The corporate-document register is unambiguous. The three pairs below illustrate the most common patterns that fail review — and the rewrite that survives it.

1. Conversational subtitle in a heading

Before · banned

Python version selection: the EOL clock is the deciding factor

Conversational subtitle. Editorial filler. Restates the body in the heading.

After · correct

Python version selection

Plain noun phrase. The body argues the EOL clock; the heading names the topic.

2. First-person narrator aside

Before · banned

Let me walk you through the rollout plan. We'll start by looking at the pilot cohort, and then I want to show you the wave-1 entry criteria. It's worth noting that the order matters here.

After · correct

The rollout proceeds in four waves. The pilot cohort consists of five IT users; wave 1 admits fifty engineering and R&D users on the pilot's exit criteria. Wave order is dependent — each wave's entry criteria are the previous wave's exit criteria.

3. "Deep dive" / decorative emoji / exclamation

Before · banned

TL;DR: the installer is unsigned! 🚀 Let's dive deeper into what that means for the rollout — and here's the kicker — SmartScreen will warn the user on every first install.

After · correct

Summary: the installer is unsigned. SmartScreen warns the user on the first install. The runbook section in the admin guide covers the dismissal steps and the silent-install bypass for Intune-managed endpoints.

Plate C — the read-aloud test in action. Every rewrite passes the test "would this survive read-aloud at a board meeting?" Every before-pattern fails it.
