# Tone and voice

Documents issued under this system are corporate communications — internal standards, technical guides, audit-grade specifications, policy memos, customer briefs. They are read by engineers, auditors, customers, and regulators. The register matches.

## Required register

Direct, declarative, third-person. State the position, then the rationale. Active voice for actions; passive only when the actor is irrelevant or an established corporate process. Specific over generic — version numbers, dates, file paths, policy clauses, named systems.

Plain noun-phrase headings: "Python version selection", "Document furniture", "Migration from v1.0". Never colon-suffixed conversational subtitles ("Python version selection: in a teacher's voice", "Document furniture: the EOL clock is the deciding factor").

## Banned patterns

The patterns below appear in machine-generated draft text and in junior business writing. They do not appear in Deccan documents.

### 1. Conversational subtitles in headings

| Banned | Correct |
|---|---|
| `## Python version selection: in plain English` | `## Python version selection` |
| `## The migration: what changes and why` | `## Migration` |
| `## Document furniture: the rules` | `## Document furniture` |

### 2. First-person narrator asides

| Banned | Correct |
|---|---|
| "Let me walk you through the rollout plan." | "The rollout proceeds in four waves." |
| "I want to show you the colour tokens." | "The colour tokens are listed in §1.3." |
| "We'll start by looking at the type stack." | "The type stack is OS-native." |

### 3. Filler openings

| Banned | Correct |
|---|---|
| "Here's the thing: the installer is unsigned." | "The installer is unsigned." |
| "It's worth noting that SmartScreen will warn the user." | "SmartScreen warns the user on first run." |
| "Let's start by examining the deployment options." | "Three deployment options are supported." |

### 4. Casual transitions

| Banned | Correct |
|---|---|
| "That said, the green is reserved." | (Use a section break or restate directly.) "Deccan Green is reserved for sustainability content." |
| "With that out of the way, on to the spatial system." | (Use the heading.) "## Spatial system" |
| "Now, let's look at the print rules." | "## Print rules" |

### 5. Decorative emoji

Banned in body content. Use words: "Required", "Prohibited", "Note", "Warning".

### 6. Exclamation marks

Banned in body prose. Use a period.

| Banned | Correct |
|---|---|
| "The installer is unsigned!" | "The installer is unsigned." |
| "Don't use the Aptos chain!" | "The Aptos chain is retired. Do not reference it." |

### 7. "Deep dive" / "let's dive in" / "rabbit hole" / "TL;DR"

Banned. Use plain language.

| Banned | Correct |
|---|---|
| "TL;DR: the system is OS-native." | "Summary: the system uses an OS-native type stack." |
| "Let's dive deeper into the colour rules." | "## Colour rules" or "The colour rules are stricter than the type rules. Three principles apply." |
| "A deep dive on Segoe UI Variable." | "Segoe UI Variable in detail." |

### 8. Mental-exercise framings

| Banned | Correct |
|---|---|
| "Imagine if every Deccan document looked different." | "Without a standard, every document re-litigates the same decisions." |
| "Think of the type stack as a fallback chain." | "The type stack is a fallback chain." |
| "A useful way to think about this is…" | (State the idea.) Or label explicitly: "Analogy: …" |

### 9. Self-reference to the document

| Banned | Correct |
|---|---|
| "This guide will teach you the colour tokens." | (Remove the sentence; the tokens are in §1.3.) |
| "By the end of this section you'll know how to install the system." | (Remove the sentence; the install steps follow.) |
| "In this document we cover…" | (Use the table of contents.) |

### 10. Em-dash narrator interjections

| Banned | Correct |
|---|---|
| "The installer is unsigned — and here's the kicker — SmartScreen will warn the user." | "The installer is unsigned. SmartScreen warns the user on first run." |
| "The migration is straightforward — bear with me — three files change." | "The migration changes three files." |

## The read-aloud test

If a sentence would feel out of place in a board paper, an audit response, or a regulator submission, rewrite it.

The corporate-document register is unambiguous; the test is fast. A sentence that would not survive read-aloud at a board meeting does not survive review.

## Allowed variations

- **Internal-only documents** (engineering specs, policy memos, IT-team-facing runbooks) may use slightly more direct phrasing and assume more shared context. The ban-list still applies.
- **Customer-facing documents** (customer letters, technical proposals, audit responses) tighten further: no contractions, full noun phrases, no jargon that requires context.
- **Sustainability submissions** (BRSR / ESG / regulatory) follow the framework's required format; the tone rules still apply where the format allows prose.

## Plain-English replacements for jargon

| Jargon | Plain |
|---|---|
| "Leverage" (verb) | "Use" |
| "Synergies" | (Remove. Be specific about what shared work.) |
| "Going forward" | "From this version onward" or remove. |
| "At the end of the day" | (Remove.) |
| "Pivot" | "Change" (or "shift", named explicitly). |
| "Reach out" | "Contact" |
| "Touch base" | "Discuss" or "follow up" |
| "Circle back" | "Follow up" or "review later" |

## Specific Deccan-preferred phrasing

From the marketing site cross-check, the corporate-canonical phrases are:

- "Custom chemical manufacturing"
- "Exclusive partnership" (the customer-engagement model)
- "Active ingredients"
- "Formulation plants" (the customer side of the manufacture chain)
- "Privately owned globally operating company"
- "QHSE" (Quality, Health, Safety, Environment)
- "AEO-T1" (Authorized Economic Operator – T1, the customs designation)

Use these in customer-facing documents and policy memos where applicable. The vocabulary is canonical; it does not need to be re-invented per document.

## What is not banned (informational)

- Em-dashes used to set off a clause. The ban is on narrator interjections, not on the punctuation itself.
- Bullet lists. The ban is on conversational openers ("Let's look at…"), not on list structure.
- Diagrams and figures. The ban is on textual filler, not on visual content.
- Specific named systems and version numbers. The register requires specificity; named systems are part of that.
